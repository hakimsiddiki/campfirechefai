# Why you only have 2 signups

Analytics (last 23 days): **0 visitors, 0 pageviews**. Semrush: the site has **no presence in Google search**. Getting 2 signups with literally no measurable inbound traffic is actually great conversion — the problem is reach, not the product.

This plan does two things in parallel:
1. **Fix the landing page** so when traffic does arrive it converts.
2. **Lay the SEO foundation** so Google can start sending free traffic over the next 4–12 weeks.

(I won't touch business logic — only the marketing surface.)

## Part 1 — Landing page & conversion fixes (`src/pages/Index.tsx`, `index.html`)

Goal: make the value obvious in 5 seconds and let people try before signing up.

1. **Sharper hero copy** focused on the exact user pain:
   - H1: *"Plan camping & RV meals in 60 seconds — no fridge, no stress."*
   - Sub: *"AI meal plans, ingredient lists, and packing tips for hikers, van-lifers and weekend campers."*
   - Primary CTA: **"Plan my first trip — free"** → `/planner`
   - Secondary: **"See a sample plan"** (opens a modal with a pre-generated 2-day plan + PDF preview)
2. **Try-before-signup**: allow ONE free plan generation without auth (rate-limited by IP in the existing edge function). Sign-up gate appears only on the second generation or PDF download. This is the single biggest conversion lever for tools like this.
3. **Social proof + trust band**: "Used by hikers, van-lifers, weekend campers" with simple icons. Add 2–3 short testimonial placeholders the user can fill in later.
4. **Visible feature grid** with screenshots of: planner output, chat assistant card, PDF export.
5. **Clear pricing section on home page** (not only `/pricing`) with the cheapest plan visible above the fold on mobile.
6. **FAQ section** targeting search-intent questions (doubles as SEO content): "What food should I take camping?", "How do I plan meals for an RV trip?", "Best no-fridge camping meals", etc.
7. **Sticky mobile CTA bar**: "Plan my trip — free" pinned bottom on mobile.

## Part 2 — SEO foundation

Semrush data shows realistic, winnable keywords (KD 22, low competition):
`easy camping meals` (12.1K/mo), `camping food ideas` (8.1K/mo), `camping meal ideas` (5.4K/mo), `backpacking meals` (5.4K/mo), `camping meal planner` (110/mo, KD 22 — easy win).

Foundation work:
1. **`index.html` head**: real `<title>`, meta description, `og:title/description/url`, canonical, and `Organization` + `WebSite` JSON-LD pointed at the live URL.
2. **Per-route head** via `react-helmet-async` on `/`, `/planner`, `/pricing`, `/auth` with unique titles + descriptions.
3. **`public/robots.txt`** allowing all crawlers + sitemap link.
4. **`public/sitemap.xml`** generated by `scripts/generate-sitemap.ts` (predev/prebuild hook) listing all public routes.
5. **Blog scaffold** (`/blog` + `/blog/[slug]`, static MDX or simple data file — no CMS needed) seeded with 3 starter posts targeting the easy keywords above. Each post has Article JSON-LD and a CTA to `/planner`.
6. **Switch the primary domain to Vercel** (`campfirechefai.vercel.app`) or, much better, a real custom domain like `campfirechef.ai`. Lovable subdomains rank poorly; if a `.vercel.app` is already where users land, point canonical/og:url and sitemap there so SEO equity isn't split.

## Technical details

- New deps: `react-helmet-async` (already covered in head-meta knowledge).
- Edge function `plan-meals`: add an anonymous-tier branch (1 plan per IP per 24h) using a small `anon_usage` table with RLS denying client access; only the edge function (service role) reads/writes it.
- New table `anon_usage(ip_hash text primary key, last_plan_at timestamptz)` — RLS enabled, no policies for anon/authenticated, full grants only to `service_role`.
- New routes: `/blog`, `/blog/:slug`. Add to `src/App.tsx` and `scripts/generate-sitemap.ts`.
- No changes to auth, payments, or the existing planner logic.
